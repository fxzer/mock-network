# 📂 源代码目录 (Source Code)

这里是 MockNetwork 项目的"心脏"，所有的核心代码都住在这里。

如果你是第一次接触 Chrome 扩展（Chrome Extension），可能会觉得有点晕。别担心，我们可以把这个扩展想象成一家**"网络快递拦截公司"**。

## 🏢 部门结构

`src` 目录下的每个子文件夹代表公司的一个关键部门：

| 文件夹                                   | 这里的代码是做什么的？ | 通俗理解 (角色与职责)                                                                                                                    |
| :--------------------------------------- | :--------------------- | :--------------------------------------------------------------------------------------------------------------------------------------- |
| **[background](./background/README.md)** | **后台服务**           | **👨‍💼 总经理 / 后台调度中心** <br> 负责管理扩展的状态（比如图标上的 ON/OFF），一直在后台默默工作，即使你没打开网页它也在。                |
| **[content](./content/README.md)**       | **内容脚本**           | **👷 驻场代表 / 桥梁** <br> 被派驻到每个网页里。它的工作是把我们的"特工"（inject）带进网页，并在网页上搭建我们的操作界面（那个侧边栏）。 |
| **[inject](./inject/README.md)**         | **注入脚本**           | **🕵️ 特工 / 拦截专家** <br> 真正的干活人！它直接潜伏在网页内部，拦截所有的网络请求（快递车），可以偷偷修改里面的货物（响应数据）。       |
| **[devtools](./devtools/README.md)**     | **开发者工具**         | **🔍 监控室** <br> 当你按 F12 打开开发者工具时，这里看到的 "M-Network" 面板就是它负责的。方便你查看和快速添加拦截规则。                  |
| **[ui](./ui/README.md)**                 | **用户界面**           | **🎨 前台接待 / 装修队** <br> 你看到的漂亮的配置界面、侧边栏、弹窗，都是这里画出来的。用的是 React 技术。                                |

## 🚀 代码是怎么跑起来的？

1. **Manifest (清单)**: 根目录下的 `manifest.json` 是公司的"营业执照"，告诉浏览器在这个 `src` 里，谁是经理（background），谁是驻场（content），谁是前台（ui）。
2. **加载顺序**: 当你打开一个网页：
   - `content` (驻场代表) 先进场。
   - 它把 `inject` (特工) 塞进网页。
   - `inject` 开始监听网络请求。
   - 你点击扩展图标，`background` (经理) 告诉 `content` 把 `ui` (前台界面) 展示出来。
